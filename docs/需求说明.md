# 广西旅游网站后台管理系统需求说明

## 1. 项目背景

为了支持广西旅游网站的日常运营和管理，需要开发一套功能完善的后台管理系统，实现对景点信息、旅游攻略、用户留言的实时维护、审核和数据分析，确保网站内容的时效性、准确性和合规性。

## 2. 核心目标

- 搭建Web后台管理系统，实现景点信息、旅游攻略、用户留言的实时增删改查/上下架/审核
- 保障数据时效准确，支持内容合规审核
- 提供多维数据统计分析，适配管理员日常运营需求

## 3. 功能模块

### 3.1 权限与基础管理
- **管理员账号**：登录/登出、密码重置、角色权限（超级管理员/内容审核员/数据查看员）
- **系统日志**：操作记录、登录日志、内容修改日志，可查询导出

### 3.2 景点信息管理
- **信息维护**：新增/编辑/删除/上下架，字段含名称、地址、等级、分类、门票、开放时间、图文详情、交通、联系方式
- **实时更新**：编辑后前端同步生效，支持批量导入/导出、按关键词/分类/状态筛选
- **关联管理**：绑定对应攻略、留言，一键查看关联内容

### 3.3 旅游攻略管理
- **内容维护**：新增/编辑/删除/审核，字段含标题、作者、关联景点、行程、图文、标签、发布状态
- **审核流程**：待审核/已通过/已驳回，驳回附原因，支持批量审核、按景点/状态/时间筛选

### 3.4 用户留言管理
- **留言处理**：查看、回复、删除、隐藏、举报处理，支持按景点/攻略/用户/时间筛选
- **合规审核**：自动关键词过滤+人工审核，屏蔽违规内容，记录审核结果

### 3.5 数据统计与分析
- **核心统计**：景点访问量、攻略阅读/收藏/点赞、留言数量/互动率、用户活跃度
- **可视化**：折线/柱状/饼图展示，支持按时间/分类筛选，数据导出（Excel/CSV）
- **数据维护**：手动/自动备份、恢复，保障数据安全

### 3.6 通用操作
- 全局搜索、分页、批量操作、富文本编辑、图片上传/压缩/裁剪

## 4. 非功能要求

### 4.1 性能
- 页面加载≤2s
- 列表查询≤1s
- 支持100+并发操作

### 4.2 安全
- 防SQL注入/XSS/CSRF
- 敏感数据加密
- 操作鉴权

### 4.3 体验
- 响应式布局适配PC
- UI简洁易用
- 操作提示清晰

### 4.4 拓展
- 模块化设计
- 便于后续新增功能
- 对接前端接口

## 5. 技术栈

### 5.1 前端
- Vue3 + Element Plus + Axios + ECharts

### 5.2 后端
- Node.js + Express + Sequelize ORM

### 5.3 数据库
- MySQL

## 6. 交付标准

- 功能无遗漏，逻辑无bug，满足实时维护、审核、分析核心场景
- 代码可读性/可维护性强，文档清晰，可直接部署迭代
